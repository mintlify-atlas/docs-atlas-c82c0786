## Overview

EverShop provides a comprehensive set of GraphQL queries for retrieving data across all modules. Queries are read-only operations that fetch data without side effects.

## Catalog Queries

### Product Queries

#### product

Retrieve a single product by ID.

```graphql
query {
  product(id: "123") {
    productId
    uuid
    name
    sku
    description
    price {
      regular {
        value
        text
      }
      special {
        value
        text
      }
    }
    image {
      url
      alt
    }
    inventory {
      isInStock
      stockAvailability
    }
  }
}
```

**Parameters:**
- `id` (ID) - Product ID or UUID

#### currentProduct

Get the current product based on the page context.

```graphql
query {
  currentProduct {
    productId
    name
    sku
    url
    variantGroup {
      variantAttributes {
        attributeName
        options {
          optionText
          productId
        }
      }
    }
  }
}
```

#### products

Retrieve a collection of products with filtering.

```graphql
query {
  products(filters: [
    { key: "status", operation: eq, value: "1" }
    { key: "price", operation: gteq, value: "10" }
  ]) {
    items {
      productId
      name
      sku
      price {
        regular {
          value
          text
        }
      }
    }
    total
    currentPage
    currentFilters {
      key
      operation
      value
    }
  }
}
```

**Parameters:**
- `filters` ([FilterInput]) - Array of filter conditions

**Filter Operations:**
- `eq` - Equal
- `neq` - Not equal
- `gt` - Greater than
- `gteq` - Greater than or equal
- `lt` - Less than
- `lteq` - Less than or equal
- `like` - Pattern match
- `nlike` - Not like
- `in` - In array
- `nin` - Not in array

#### productSearch

Search for products.

```graphql
query {
  productSearch {
    keyword
    products {
      items {
        productId
        name
        sku
      }
      total
    }
  }
}
```

## Query Best Practices

<AccordionGroup>
  <Accordion title="Request Only What You Need">
    GraphQL allows you to request exactly the fields you need. Avoid requesting unnecessary data:
    
    ```graphql
    # Good - Only request needed fields
    query {
      product(id: "123") {
        name
        price { regular { value } }
      }
    }
    
    # Avoid - Requesting all fields when not needed
    ```
  </Accordion>

  <Accordion title="Use Variables for Dynamic Queries">
    Use variables instead of inline values:
    
    ```graphql
    query GetProduct($id: ID!) {
      product(id: $id) {
        name
        sku
      }
    }
    ```
    
    Variables: `{ "id": "123" }`
  </Accordion>

  <Accordion title="Leverage Fragments">
    Use fragments to reuse field selections:
    
    ```graphql
    fragment PriceFields on Price {
      value
      currency
      text
    }
    
    query {
      product(id: "123") {
        price {
          regular { ...PriceFields }
          special { ...PriceFields }
        }
      }
    }
    ```
  </Accordion>

  <Accordion title="Handle Pagination">
    Use filters and limit results for large collections:
    
    ```graphql
    query {
      products(filters: [
        { key: "page", operation: eq, value: "1" }
        { key: "limit", operation: eq, value: "20" }
      ]) {
        items { name }
        total
        currentPage
      }
    }
    ```
  </Accordion>
</AccordionGroup>
